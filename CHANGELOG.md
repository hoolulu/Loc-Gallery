# Changelog

## [4.4.0] - 2026-07-24

**启动与缩略图**：重启后进度条秒开、格式筛选、白天模式与截图兜底。

### 新增

- **格式筛选**：顶栏按播放格式（标准 H.264、HLS、可修复碎片化 MP4 等）筛选；后台 `format_index` 持久化索引，前台只读
- **白天模式**：设置与顶栏切换 `ui_theme`（dark/light），主题写入服务器配置并同步 localStorage
- **缩略图进度条模式**：设置项 `thumb_progress_bar`（auto / always / never）；auto 下顶栏「缩略图」芯片可临时收起/展开，翻页或刷新后恢复默认

### 改进

- **启动加速**：仅同步扫描当前库，其余库与缩略图索引同步放后台；进度统计直接读 `index.json`，不再逐个 stat 磁盘文件；前端打开即拉取进度
- **缩略图截图兜底**：主位置解码失败时依次尝试 30s/15s/5s 及多档比例，修复短视频在默认 70% 处损坏导致永久失败
- **悬停预览**：大图角标显示格式类型；白天模式下播放检测弹窗、路径提示等对比度优化

### 修复

- 缩略图队列/索引重建时的死锁（持锁调用 `_has_usable_thumb`、`_file_matches` 等）
- 主题设置被旧 localStorage 覆盖、与服务器 `ui_theme` 不一致
- 手动随机重生成缩略图未使用设置中的 `thumb_random_min/max`

## [4.3.0] - 2026-07-23

**播放与缩略图体验**：连播更省盘、播放列表缩略图可原地刷新、进度条不再顶开页面。

### 改进

- **连播切歌**：先断开 HLS/Range 拉流并停止切片，再播下一首；剩余不足 3 分钟时预取下一首 `play/info`，减轻机械盘风暴
- **缩略图进度条**：改为顶栏下方浮层显示，展开/收起不再推动页面布局；点击页面其他区域可收起

### 修复

- 播放页右侧列表右键「生成缩略图」后，对应单元格可正确显示排队/生成中并刷新结果（不再依赖主网格 `thumb-{id}`）

## [4.2.0] - 2026-07-22

**修复与入库**：批量 remux 不再逐文件开关监视器；新下载完成后恢复自动入库与缩略图排队。

### 新增

- **`POST /api/remux/batch/begin`** / **`batch/end`**：批量修复期间按库暂停文件监视，整批结束后统一恢复并补扫索引
- 新文件稳定后 **`upsert_video_from_path`** 增量入库（无需等全库扫描）

### 改进

- 文件监视器识别 **`.part` → `.mp4` 重命名**（`dest_path`），下载完成可正确触发入库
- remux 监视策略改为 **库级会话**（单任务计数 + 批量会话），避免批量修复中间反复全库刷新

### 修复

- 4.1.0 remux 屏蔽误伤 `notify_file_activity`，导致新下载视频长期不入库
- 批量修复「修一个恢复监视、再抑制」易引发机械盘风暴与服务假死

## [4.1.0] - 2026-07-22

**碎片化 MP4 修复**：重写 remux 流程，修复完成后可原地去角标并直连播放，不再拖垮整站。

### 新增

- **可修复角标**与右键/批量 **「修复为标准 MP4」**（流复制 + faststart，仅碎片化 H.264）
- remux 期间 **屏蔽文件监视器全库扫描**，避免机械盘长时间 100%
- 修复后 **`seed_direct_playback_plan`**：直接写入 `direct` 播放计划，跳过 ffprobe 重探

### 改进

- remux 流程：**改名 `.bak` → 写出原路径 → 恢复时间戳**（无大文件 replace）；`.bak` 后台 `unlink`
- ffmpeg 完成即 **`done`**；前端 **原地去掉角标**，不必整页刷新
- `/api/play/info` 等播放探测改 **`asyncio.to_thread`**，避免阻塞事件循环

### 修复

- 修复收尾阶段回收站删 `.bak` / 全库 `refresh_cache` 导致 overlay 假卡死与整站无响应
- 修复后点击播放仍触发冷启动探测卡死整站的问题

## [4.0.0] - 2026-07-21

**稳定版**：部分站点源（多段小 `mdat` MP4）识别与 HLS 播放、机械盘友好的切片调度与拖进度续切。

### 新增

- **多段 mdat 交错 MP4** 检测（播放计划 v15）：误判「标准格式」的 NJAV 类源自动 **HLS 边切边播**
- HLS **切片水位**：盘上领先约 **2 分钟** 挂起 ffmpeg、约 **1 分钟** 恢复；近边 **90 秒** 强制续切
- **`POST /api/play/catchup/{video_id}`**：拖进度 / 卡顿 / 贴近已切末尾时续切；大幅超前时 **`-ss` + append** 从目标处续切
- 播放状态返回 **`produced_end_sec`**、**`segment_seconds`**

### 改进

- 切片控制 **仅依据磁盘已切进度**，不再被浏览器旧缓冲误导
- ffmpeg 子进程 **低于正常优先级**（Windows），减轻与其它任务抢盘
- 直连加载文案区分 **「文件约 N MB」** 与缓冲进度；覆盖层显示 **多段交错 MP4**

### 修复

- 多段小 `mdat` 被标为标准 H.264 直连导致机械盘长时间 100% 与起播极慢
- HLS 拖到最后已切分片附近需长时间等待下一段

## [3.3.0] - 2026-07-21

**播放列表与设置**：列表分页加载、滚轮快进可配置、设置页紧凑布局。

### 新增

- 播放页右侧列表 **加载下一页**（滚到底自动加载 + 按钮）；范围跟列表筛选，排序跟播放列表下拉（「列表顺序」= 网格 sort + 从当前页续载）
- 连播/点「下一个」到末尾时 **自动拉取下一页**；加载下一页 **不抢滚动位置**
- 刷新或切换排序后 **定位到正在播放的条目**（必要时自动补拉页）
- 设置 **播放区滚轮快进/快退（秒）**（0 = 关闭）；画面区滚轮下/上 ±N 秒

### 改进

- **全局设置** 改为流式紧凑布局，控件按内容宽度，视频库路径列不再撑满弹窗

### 修复

- 播放列表分页后点击缩略图无反应（`playVideoHtml5` 缺少 `base` 变量）
- 已在播放页内换片时误重置播放列表为仅当前网格页

## [3.2.0] - 2026-07-21

**播放页与标题**：原生控件回归、列表右键删除连播、卡片标题智能截取。

### 新增

- 设置 **缩略图进度条**：自动 / 始终显示 / 从不；自动收起时顶栏红色底线，**缩略图** 芯片可点击展开/收起详情
- 播放页 URL **`?play=`** 刷新后可恢复当前视频；首页 **`/`** 与 **`?view=favorites|history`** 导航
- 播放列表 **右键菜单** 与网格一致（收藏、重命名、移动、删除等）

### 改进

- 播放器恢复 **原生 `<video controls>`**，去掉自定义进度条与壳层全屏等实验改动
- 播放页顶栏标题/路径/按钮排版与 **6px 圆角** 统一按钮样式
- 卡片标题：无番号时截到 **`(年份)`** 右括号；无括号按字数截取；**`XXX-NNN`** 番号规则不变
- 点 **刷新** 时重载标题规则并 **重绘网格**；扫描逻辑每次刷新重载 `title` 模块

## [3.0.0] - 2026-07-20

**稳定版**：HTML5 直连读盘与播放页体验集中修复，推荐升级。

### 新增

- **`/api/stream` 小块流式传输**（`range_stream.py`）：256KB 分块 + 客户端断开即停读盘，替代整段 `FileResponse` 全速抽读
- 设置项 **HTML5 播放列表连播**、**HTML5 记忆播放位置**（可关闭）
- `/api/play/info` 返回 **续播进度**（`playPosition` / `playDuration`）

### 改进

- 播放时磁盘读取接近 **码率级缓冲**（约 2–3 MB/s），不再长时间 100% 猛读
- 退出/切换时 **`hard` 重建 `<video>`**，尽快取消浏览器 Range 请求
- 关页 `pagehide` 用 **keepalive** 保存续播进度
- 续播 **seek 等待 `seeked`**；播放页内点列表 **立即切换**（先递增 `playSession`，进度异步保存）
- 播放列表 **事件委托**，避免重绘后点击失效

### 修复

- 中文文件名 `Content-Disposition` 导致 `/api/stream` 500、全部「视频加载失败」
- 播放页右侧列表切换无反应
- 连播 `ended` 在重建 video 节点后丢失
- 记忆播放位置未从服务端刷新时不续播

## [2.2.0] - 2026-07-20

### 新增

- **碎片化 MP4 修复**：流复制重封装为标准 MP4（faststart）；播放信息返回 `remux_reason`；支持批量「修复为标准 MP4」
- 缩略图 **「非标准」角标**（碎片化 / 伪装 TS）
- 碎片化且策略为「本地播放器」时，弹窗选择 **PotPlayer** 或 **修复**

### 改进

- **默认播放方式改为 HTML5**；修复曾将 `html5` 误读为 `smart` 导致仍走 PotPlayer 的问题
- 播放计划 **v13**：前部扫描 `moof` 以识别碎片化 MP4；策略标签含版本号便于缓存失效
- **退出播放页** 加强断流（HLS `stopLoad`、直连取消、切片任务停止），减轻退出后持续读盘
- remux 替换原文件时 **占用重试**，大文件备份提示更明确
- AV1/HEVC/VP9、伪装 TS、大文件等恢复 **自动 HLS/转码**；仅碎片化 `external` 弹窗

### 修复

- PotPlayer 模式下点击卡片可正常调起外部播放器

## [2.1.0] - 2026-07-20

### 新增

- **HTML5 续播**：播放进度写入 `play_history.json`（`position_sec`），下次打开同一视频从上次位置继续（直连与 HLS；HLS 通过 hls.js `startPosition`）
- **暂停播放时挂起 ffmpeg 切片**（Windows 进程挂起），继续播放时恢复；切换视频、返回列表、关闭页面仍停止切片进程
- API：`POST /api/history/position` 保存进度；`POST /api/play/pause` / `POST /api/play/resume` 控制切片进程挂起/恢复

### 改进

- HLS 片段时长 **6 秒 → 30 秒**，显著减少小文件数量与机械硬盘 I/O
- 去掉 HLS `temp_file` 标志，避免每段双写
- 切片缓存元数据增加 `segment_seconds`，调整片段参数后旧缓存自动失效并重新切片

### 修复

- 移除强制将播放头归零的逻辑，修复「记忆播放位置」无效的问题
- **PotPlayer 默认播放**：点击缩略图无反应；改为调用 `/api/play-external/`，并与合并后的播放设置同步
